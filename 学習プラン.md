# 国内旅行業務取扱管理者試験 学習システム構築プラン

## Context（なぜこれをやるか）

- **試験方式: CBT**（コンピュータ受験）。試験日は **令和8年（2026年）9月3日(木)〜9月25日(金)** の期間内で**受験者が選択した日時**。
- 起点は 2026-06-16。受験日までは **残り約11〜14週**（選ぶ日による）。当初想定の16週より短いため、本プランは**12週の核 + 直前バッファ**に圧縮して設計する。
- **推奨: CBT枠を早めに予約し、9月中旬（例: 9/17前後）を狙う**。早すぎると仕上がらず、ぎりぎり9/25にすると体調・予定の予備日が無くなるため、中旬で予備日を残すのが安全。
- 現状は **ほぼ初学者**。1日に確保できるのは **30分〜1時間**。
- 手元には R03〜R07 の5年分の過去問（問題PDF + 解答PDF）がある。これを「ただ解く」のではなく、**構造化・分析・反復・即時添削**できる学習システムに作り変えることで、限られた時間で合格ラインに最短到達するのが狙い。
- 合格基準は **3科目すべて60点以上**（1科目でも足切りで不合格）。よって「得意科目で稼ぐ」戦略は通用せず、**全科目を満遍なく60点超に乗せる**設計が必須。

### 試験構成（過去問から確認済み）
| 科目 | 範囲 | 配点 | 性質 |
|------|------|------|------|
| 1 | 旅行業法及びこれに基づく命令（問1-25） | 4点×25=100 | 条文の暗記＋理解。頻出論点が固定的で得点源にしやすい |
| 2 | 旅行業約款・運送約款・宿泊約款（問26-50） | 4点×25=100 | 約款の細部暗記。1と並んで対策効果が高い |
| 3 | 国内旅行実務（問51-85） | 4点×12＋2点×26=100 | JR運賃・料金計算（ルール）＋観光地理（暗記）。初学者が最も苦戦する科目 |

問題形式は四肢択一（ア〜エ）。単一選択と「すべて選びなさい」の複数選択が混在。PDFはテキスト抽出可能。

> **v2（Codexレビュー反映済み）**: 「立派な学習システム」より「すぐ回る軽量運用」を優先。教材整備に時間をかけすぎない／科目を直列にせず混ぜる／足切り対策で論点を層化、の3点を反映した。

## 構築する学習システム（ディレクトリ構成）

**MVP（最初に作る最小限。これだけで勉強は回る）**
```
domtrip/
├── 過去問/                    # 元PDF（変更しない）
├── 問題集/                    # PDFから抽出した構造化問題バンク
│   ├── 科目1_旅行業法/R03.md 〜 R07.md
│   ├── 科目2_約款/R03.md 〜 R07.md
│   └── 科目3_国内旅行実務/R03.md 〜 R07.md
└── 学習記録/
    ├── 誤答ログ.md            # 論点単位＋ミス種別で記録→間隔反復で再出題
    └── 週次計画.md            # 16週スケジュール＋日々のログ
```

**後付け（演習しながら必要になった分だけ作る。最初から作り込まない）**
```
├── 分析/出題傾向.md           # 論点頻度のA/B/C層化（演習が進んでから集計）
└── ノート/要点メモ.md         # 誤答が増えた論点だけ要点化（先に全部は作らない）
```

各問題集ファイルの1問あたり形式（アクティブリコール用に問題と解答・解説を分離。**解説は必ず根拠に紐付け**）:
```markdown
### 問1 [論点: 旅行業法の目的]
次の記述のうち、法第1条「目的」に定められているものをすべて選びなさい。
- ア. …  / イ. …  / ウ. …  / エ. …
<details><summary>解答</summary>正解: ア・エ ／ 根拠: 旅行業法第1条 ／ 解説: …（なぜ他が誤りか）</details>
```
- 業法 → 根拠条文 / 約款 → 該当約款の項目 / 実務計算 → 計算手順テンプレ、のいずれかに必ず紐づける。

誤答ログの1件あたり形式（**問題単位でなく論点単位**。問題暗記を防ぎ類題へ転用できる）:
```markdown
| 日付 | 出典 | 論点 | ミス種別 | 正しい根拠 | 次回復習日 |
|------|------|------|----------|-----------|-----------|
| 6/16 | R07問3 | 登録申請先 | 知識不足 | 第3種=都道府県知事 | 6/17 |
```
- ミス種別 = 知識不足 / 読み違い / 計算ミス / 選択肢比較不足（改善の打ち手が変わるので必ず分類）。

## 採用する勉強法（「最強」の中身）

データを活かして以下4つを組み合わせる。いずれも学習効果が実証されている手法:

1. **アクティブリコール** — 僕が過去問を出題→ユーザーが答える→即採点。さらに**解答を見る前に「なぜその選択肢が誤りか」を口頭で言わせる**（再認でなく想起を増やす）。
2. **間隔反復（spaced repetition）** — 誤答を `誤答ログ.md` に記録し、翌日・3日後・1週間後と日を空けて再出題。
3. **インターリービング** — 1科目を長期間連続でやらず、主軸科目に他科目を細く混ぜる。忘却を防ぎ保持率を上げる。
4. **出題傾向ドリブン（A/B/C層化）** — 論点を A:頻出で必ず取る / B:60点到達に必要な中頻度 / C:低頻度の捨て候補 に3層化。頻出だけに寄せず、足切り回避のため全範囲に薄くは触れる。
5. **即時フィードバック** — その場で根拠（条文・約款項目・計算手順）に紐づけて解説。

### 日々の学習ループ（採点・記録の時間も込みで再設計）
**平日 30〜45分**
- 5分: 前回の誤答2〜3問を復習
- 20〜25分: 新規過去問演習（僕が出題→採点→根拠解説。チャットで「クイズ」と言えば開始）
- 10分: 解説確認と誤答ログ記録

**休日・余裕のある日 60分**
- 10分: 誤答復習 / 30分: 新規演習 / 20分: ミニまとめ or 科目3の計算練習

毎日インプット枠を固定で取らず、過去問から必要知識を拾う形で進める。**ただし科目3の運賃料金計算だけは別枠で基礎整理が要る**。

### 12週スケジュール（CBT 9月受験・インターリービング＋足切り対策版）
受験日を **9月中旬（9/17前後）** に置いた場合の目安。9/25を選ぶなら各フェーズに+数日の余裕、9月初旬を選ぶなら週11-12を前倒しして圧縮する。

| 週（目安） | 主軸 | 混ぜる（細く維持） |
|----|------|------|
| 1（6/中） | 全科目の全体像把握＋科目1開始 | 科目3の計算ルールだけ先行接触 |
| 2-4（6末-7月） | 科目1 旅行業法 | 週2回 科目2 or 3 を混ぜる |
| 5-7（7末-8月） | 科目2 約款 | 科目1の誤答復習＋週1回ミニ実務。**週6で短縮版の通し演習を一度**入れる |
| 8-10（8月-9初） | 科目3 国内旅行実務（運賃料金計算＋観光地理） | 科目1・2を薄く維持＋週末に年度別ミニ通し |
| 11（9/中） | 全科目 年度別通し演習（**CBT想定で本番同様に時間計測**） | 時間配分調整＋足切り危険科目の底上げ |
| 12（受験直前） | 誤答トップ論点潰し＋暗記・計算・地理の最終反復 | — |

科目1・2を先に主軸化する理由: 暗記中心で短期に60点超へ乗せやすく、合格の土台（足切り回避）を早期に確保できるため。ただし完全直列にせず科目3を週1から薄く先行させ、後半の負荷集中と前半科目の忘却を防ぐ。

> **CBT特有の注意**: 紙でなく画面上で解くため、計算問題のメモは配布されるメモ用紙で行う。通し演習（週11）は画面で問題を見てメモ用紙に計算する形を再現しておくと本番で慌てない。

## 実行ステップ（承認後の作業順）

**Step 1（今セッション）: MVPだけ作って即・勉強を始められる状態にする**
1. `問題集/` に **R07の科目1を抽出**（PDF→構造化Markdown、根拠付き解説）。1科目分で形式を確定させてから横展開する。
2. `学習記録/誤答ログ.md`（論点単位＋ミス種別テンプレ）と `学習記録/週次計画.md`（16週スケジュール）を作成。
3. CLAUDE.md に「クイズモードの起動方法」と本システムの使い方を追記。
4. その場で**科目1のミニクイズ（5問）を実際に出題・採点**し、ループが回ることを確認する。

**Step 2以降（後続セッション、スケジュールに沿って）**
- 残り（R07科目2・3 → R03-R06）の問題集抽出を順次。
- `分析/出題傾向.md`（A/B/C層化）は演習が一定量たまってから集計。
- `ノート/要点メモ.md` は誤答が増えた論点だけ後付けで要点化。
- 毎回の学習セッションで「クイズ」起動→出題・採点・誤答ログ更新。

## 検証（システムが機能していることの確認方法）
- Step 1完了後、その場で**科目1のミニクイズ（5問）を実際に出題・採点**し、ループが回ることを確認する。
- 間違えた問題が `誤答ログ.md` に論点単位＋ミス種別＋次回復習日つきで記録され、次回出題対象になることを確認する。
- 各問の解説が根拠（条文/約款項目/計算手順）に紐づいていることを確認する。
