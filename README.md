# 国内旅行業務取扱管理者試験 学習システム

過去問（R03〜R07）を構造化し、**完全オフラインで動く間隔反復クイズアプリ**・**進捗ダッシュボード**・**暗記シート**で合格（3科目すべて60点以上）を目指す自己学習システム。

## 公開ページ（GitHub Pages）
- 📱 **学習アプリ（クイズ）**: <https://kasei-san.com/domtrip/>
- 📊 **ダッシュボード**: <https://kasei-san.com/domtrip/dashboard.html>
- 📝 **暗記シート（通勤用）**: <https://kasei-san.com/domtrip/cram-sheet.html>

## 何があるか

| ファイル / フォルダ | 役割 |
|---|---|
| `index.html` | 過去問クイズアプリ（4択・解答リビュー・暗記カード・苦手復習）。進捗は localStorage に保存 |
| `dashboard.html` | 到達度・正答率・周回・連続日数・カウントダウンを表示。アプリと同じ進捗をライブ集計 |
| `app_data.js` | アプリが読む問題データ（`tools/build_app_data.py` が生成・編集禁止） |
| `sw.js` / `manifest.webmanifest` / `icon.svg` | PWA用（ホーム画面追加でオフライン動作） |
| `questions/` | 過去問を構造化した Markdown（科目1旅行業法 / 科目2約款 / 科目3国内旅行実務 × R03〜R07） |
| `cram-sheet.md` / `.html` | 通勤用の要点（数字・ひっかけ・観光地理 都道府県別） |
| `study-plan.md` / `app-spec.md` | 学習計画とアプリ仕様 |
| `tools/build_app_data.py` | `questions/` と `cram-sheet.md` から `app_data.js` を再生成 |
| `past-exams/` | 元の過去問PDF（問題・解答） |

## 使い方

### PC・Android（ローカル）
**ファイルをダブルクリックで開くと動かない**（`file://` ではブラウザが `app_data.js` の読み込みと localStorage をブロックするため）。簡易サーバ経由で開くこと:

```sh
cd このフォルダ
python3 -m http.server 8731
# ブラウザで http://localhost:8731/index.html を開く
```

### iPhone / ブラウザ（GitHub Pages で公開済み）
公開URL: **https://kasei-san.com/domtrip/** （`index.html` がルート）

iOS Safari は `file://` だと localStorage が使えない（進捗が保存できない）ので、必ずこの https URL から使う:

1. iPhone の Safari で <https://kasei-san.com/domtrip/> を開く
2. 共有ボタン →「ホーム画面に追加」
3. 以降はアイコンから起動。Service Worker でキャッシュされ**機内モードでも動作**、進捗も保持される

> 更新の反映: `main` に push すれば数十秒で自動反映。Service Worker のキャッシュを確実に更新したいときは `sw.js` の `CACHE`（例 `domtrip-v2`）の版番号を上げる。

### 複数人で使う場合
このアプリに**サーバ・アカウント・共有DBはない**。進捗は各自のブラウザの localStorage に保存され、**人ごと・端末ごとに完全に独立**する（他人の進捗と混ざらない／どこにも送信されない）。同じURLを何人で開いても問題ない。
- 注意: **同じ端末・同じブラウザを共有すると localStorage も共有**され進捗が混ざる。各自の端末／ブラウザ（または別プロファイル）で使うこと。
- 進捗を別端末へ移すときは、アプリの「⬇ JSONで保存／📋 コピー」でエクスポート → 移行先で取り込む。

## 学習の流れ

1. **学習アプリ**で問題を解く（科目・年度・モードで絞り込み可）
   - 間隔反復（固定間隔SRS: 1→3→7→16→35日）。正解で間隔が延び、不正解で翌日に戻る
   - 「本日の復習（due）」を優先し、新規は1日上限内で出題
2. **学習ダッシュボード**で到達度・正答率・連続日数を確認（アプリと相互リンクで行き来）
3. 苦手は「苦手復習」モードで集中的に
4. 通勤中は `cram-sheet.html` を流し読み

## 進捗データと AI 分析

進捗は端末内の localStorage に保存される。**エクスポートJSON**を取り出せば、Claude（このリポジトリを開いた状態）に渡して分析・質問ができる:

1. アプリの **「⬇ JSONで保存」** または **「📋 コピー」** でエクスポート
2. その JSON を Claude のチャットに貼る／ファイルで渡す
3. Claude は `app_data.js`・`questions/` と突き合わせて、**苦手論点ランキング・科目別の合格圏判定・復習すべき札・間違えた問題の解説**などを返す

> 別端末の進捗を見たいときは、ダッシュボードの「別端末の進捗を取り込む」欄にエクスポートJSONを貼る（同一サイト上のアプリ進捗は自動反映されるので不要）。

## 問題集を更新したら（再ビルド）

`questions/*.md` や `cram-sheet.md` を編集したら、`app_data.js` を再生成する:

```sh
python3 tools/build_app_data.py
```

## 注意

- **このリポジトリは公開前提。個人情報・絶対パス・秘密情報をコミットしない**（詳細は `CLAUDE.md`）。
- **学習進捗のエクスポートJSONは公開リポジトリにコミットしない**（個人データ）。履歴を残すなら `.gitignore` 済みのローカルフォルダに置く。
- 科目3は運賃計算図・地図など図表依存の設問が多く、本文は要点＋正解＋解説＋「図表は過去問PDF参照」の形（図表は再現していない）。
